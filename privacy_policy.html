Политика конфиденциальности — 4Pulse

Последнее обновление: апрель 2026

1. Какие данные обрабатываются

Расширение 4Pulse работает исключительно с данными форума 4pda.to и обрабатывает следующую информацию:

  • Куки сессии (member_id, pass_hash) — используются только для установления соединения с сервером 4pda.to и проверки авторизации. Куки читаются через Chrome Cookies API и не копируются, не передаются и не хранятся за пределами браузера.
  • Данные форума (темы, сообщения, счётчики) — загружаются с серверов 4pda.to и временно хранятся в локальном хранилище браузера (chrome.storage.local) для отображения в интерфейсе расширения.
  • Настройки пользователя (тема оформления, звуки, интервалы) — хранятся только локально в браузере через chrome.storage.local.

2. Что расширение НЕ делает

  • Не передаёт никакие данные разработчику расширения или третьим лицам
  • Не имеет собственного сервера для сбора данных
  • Не отслеживает историю просмотров и не читает содержимое других сайтов
  • Не использует аналитику, рекламные SDK или трекеры
  • Не хранит пароли или платёжные данные

3. Сетевые соединения

Расширение устанавливает соединения исключительно со следующими серверами:

  • 4pda.to и *.4pda.to — основные серверы форума для получения уведомлений
  • itunes.apple.com, musicbrainz.org, coverartarchive.org — только при использовании функции встроенного радио, для получения обложек треков
  • de1.api.radio-browser.info — только при использовании встроенного радио, для поиска радиостанций
  • fonts.googleapis.com, fonts.gstatic.com — загрузка шрифта интерфейса

4. Хранение данных

Все данные хранятся локально в браузере пользователя с использованием стандартного API chrome.storage.local. Данные не покидают устройство пользователя.

5. Контакт

По вопросам конфиденциальности: страница расширения на форуме 4pda.to или через механизм обратной связи Chrome Web Store.

Privacy Policy — 4Pulse (English)

Last updated: April 2026

1. Data processed

4Pulse extension works exclusively with 4pda.to forum data and processes:

  • Session cookies (member_id, pass_hash) — used only to establish a connection with 4pda.to and verify authentication. Cookies are read via the Chrome Cookies API and are never copied, transmitted, or stored outside the browser.
  • Forum data (topics, messages, counters) — loaded from 4pda.to servers and temporarily stored in browser local storage (chrome.storage.local) for display in the extension interface.
  • User settings (theme, sounds, intervals) — stored only locally in the browser via chrome.storage.local.

2. What the extension does NOT do

  • Does not transmit any data to the extension developer or third parties
  • Has no proprietary server for data collection
  • Does not track browsing history or read content from other websites
  • Does not use analytics, advertising SDKs, or trackers
  • Does not store passwords or payment information

3. Network connections

The extension connects exclusively to:

  • 4pda.to and *.4pda.to — main forum servers for notifications
  • itunes.apple.com, musicbrainz.org, coverartarchive.org — only when using the built-in radio feature, to fetch track artwork
  • de1.api.radio-browser.info — only when using built-in radio, to search for stations
  • fonts.googleapis.com, fonts.gstatic.com — loading interface font

4. Data storage

All data is stored locally in the user's browser using the standard chrome.storage.local API. Data never leaves the user's device.

5. Contact

For privacy questions: extension page on 4pda.to forum or via Chrome Web Store feedback.
